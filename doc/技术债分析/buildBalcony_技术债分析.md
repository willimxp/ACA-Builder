# 技术债分析报告: buildBalcony.py

## 1. 文件概览
- **文件路径**: `buildBalcony.py`
- **主要功能**: 生成栏杆、勾栏、平座等结构。
- **代码规模**: 约 936 行。

## 2. 主要技术债

### 2.1 路径跟随算法的局限性
- **问题描述**: 栏杆通常需要沿着建筑的边缘（Path）生成。代码中的路径跟随算法可能只支持矩形或简单的多边形，难以适应复杂的曲面或非正交边缘。
- **影响**: 限制了建筑形制的灵活性。

### 2.2 阵列逻辑的硬编码
- **问题描述**: 栏杆中的望柱和栏板是重复单元。代码可能使用了简单的 `for` 循环计算坐标，而不是利用 Blender 的 Array Modifier。
- **影响**: 生成的模型是破坏性的（Destructive），生成后无法通过修改修改器参数来调整间距。

### 2.3 细节层次 (LOD) 缺失
- **问题描述**: 栏杆往往包含复杂的镂空纹理。代码可能在所有距离下都生成高模。
- **影响**: 场景面数爆炸。

## 3. 改进建议
1.  **利用修改器**: 优先生成基础路径曲线，然后挂载 Array 和 Curve 修改器来生成栏杆，保持非破坏性编辑能力。
2.  **模块化设计**: 将栏杆样式抽象为“单元模式”，允许用户自定义单元 Mesh。
