# ACA插件国际化处理流程

## 概述
记录在做ACA国际化过程中的工作流程

## 1、技术栈搭建
1. 新建了i18n.py模块
    - 添加插件偏好设置中的语言选择（包括切换语言后的自动刷新）
    - 封装了_()函数，供需要国际化的代码中调用
    - 自动载入PO/MO文件（支持多翻译域合并,aca_builder,aca_xml）
2. 新建en_US/LC_MESSAGES/，存放aca_builder.po,aca_xml.po
3. 修改__init__.py，初始化i18n模块
参考:locale/i18n_spec.md

## 2、代码预处理
1. 制作i18n SKILL
   用于从代码中提取所有的中文文本，自动生成了PO文件。
   然后由AGENT进行了初步的翻译，更新PO文件，编译MO文件。
   具体参考：
   .trae/skills/aca\_i18n/i18skill\_spec.md
   这个SKILL通过Trae生成。
2. 逐一针对项目中的.py文件执行i18n SKILL，增量更新到PO文件。
3. XML文件单独进行了处理

## 3、翻译优化
### 目的
- 统一古建筑专业术语，保持全局的一致性
- 针对插件的界面显示，兼顾友好性和专业性
### 3.1 术语提取
1、基于PO文件，使用大模型进行术语的提取
```
从.po文件内容中，完成以下任务：
1. 精准识别并提取所有**中国古建筑专用术语**（仅中文），以及该术语在.po文件中对应的**英文翻译**；
2. 严格过滤非古建筑领域的通用词汇、注释、代码片段、格式字符（如msgid/msgstr标记、行号、注释符号等）；
3. 去重处理：若同一中文术语对应多个英文翻译，保留.po文件中出现的全部翻译（以数组形式存储）；
4. 分组处理：所有术语按照古建筑的结构分组，如，台基，屋身，梁架，屋顶，瓦作等。
5. 输出格式要求：仅返回标准JSON字符串，无任何多余文字、说明、注释；保存在项目根目录下。
6. JSON结构示例：
{
"斗拱": ["dougong", "bracket set"],
"庑殿顶": ["hip-and-gable roof"],
"榫卯": ["mortise and tenon joint"]
}
```
### 3.2 术语优化
你是专注于中国古建筑文献英译的专业术语工程师，精通中国古建筑营造术语、结构名称。
在当前术语表中的英文翻译中，补充更多的可选英文翻译，尽可能标准、正式、学术规范。
严格遵守以下规则：
- 使用中国建筑史与中国古建筑学界通用标准英文译法，严格禁止自创
- 优先采用：中国营造学社、梁思成、刘敦桢等权威译法，故宫博物院、国家文物局官方英译，清华大学、东南大学等院校，建筑史教材通用译法，海外权威汉学建筑著作固定译法
- 忠于原文：不增删信息，不改写原意，保留档案的严谨性。
- 文风正式：使用学术档案式英文，简洁、客观、无口语化表达。
- 输出格式要求：仅返回标准JSON字符串，无任何多余文字、说明、注释；保存在项目根目录下。
JSON结构示例：
{
"斗拱": ["dougong", "bracket set"],
"庑殿顶": ["hip-and-gable roof"],
"榫卯": ["mortise and tenon joint"]
}
